# Roadmap

## Phase 0 - Foundation
Status: In progress

Goal: Establish the product shell and the initial UX direction.

Planned work:
- Create the initial React + TypeScript app shell
- Define the core product narrative and information architecture
- Build the first landing and audit-entry experience
- Draft documentation for product, architecture, and future implementation

## Phase 1 - MVP Audit Experience
Status: Planned

Goal: Deliver a usable first scan experience for one business at a time.

Planned work:
- Business input form
- Local SEO and website audit scoring
- Review and reputation signal review
- Actionable recommendations panel
- Simple report export

## Phase 2 - Multi-Location and Saved Workflows
Status: Planned

Goal: Support agencies and operators managing multiple locations.

Planned work:
- Save and revisit audits
- Compare locations side by side
- Track changes over time
- Tag and organize client workspaces

## Phase 3 - Data Integrations
Status: Planned

Goal: Improve accuracy and reduce manual work.

Planned work:
- Connect with listing and review data sources
- Add website crawl and on-page analysis
- Support analytics and search console-style signals
- Introduce background processing for larger reports

## Phase 4 - Intelligence Layer
Status: Planned

Goal: Turn raw signals into more useful insight and automation.

Planned work:
- AI-generated recommendations
- Trend detection and alerting
- Suggested content and update plans
- Smarter competitor comparison

## Long-Term Vision
Local Signal Scanner should become a reliable operating system for local visibility management, helping businesses maintain trust, discoverability, and conversion readiness across search and AI-driven experiences.
